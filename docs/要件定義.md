# ボード型TODOアプリ 要件定義（ドラフト）

## 1. ユースケース・使い方

- シチュエーション：アイデアメモ、タスクのラフ整理などに使う
- 利用者：1人用
- 対応デバイス：スマホ対応（レスポンシブ）
- 動作環境：オンラインのみ

## 2. ボードとカードの仕様

- ボード枚数：1枚のみ（当面）
- カード情報：
  - タイトル
  - 詳細
  - 色
  - 優先度
- カードサイズ：可変
- 配置：ボード上に完全自由配置（x, y 座標指定）

## 3. ドラッグ＆ドロップ挙動

- ドラッグ中の見た目：カードを半透明にする
- ドロップ可能範囲：ボード全体
- 配置の保存タイミング：ドロップ時に即保存
- カード同士の重なり：重なりを許容する

## 4. 基本操作

- カード追加：
  - ボタンから追加
  - ボード上ダブルクリックで空カード生成
- カード編集：インライン編集
- カード削除：ゴミ箱操作で削除
- ボード操作：
  - ズーム（拡大・縮小）
  - パン（ボード全体をドラッグ移動）

## 5. データ保存・同期

- 保存先：
  - ローカルストレージ
  - 将来的にバックエンドAPI（DB）対応
- ユーザー管理：ログイン機能を持ち、ユーザーごとにボードを持つ
- 履歴機能：「元に戻す（Undo）」機能が必要

## 6. UI・UX

- テーマ：カンバン風デザイン
- ダークモード：対応する
- キーボードショートカット：
  - Enter：カード追加
  - Delete：カード削除 など
- アニメーション：
  - カード追加時にきらきらエフェクトを表示

## 7. 技術スタック・構成

- フロントエンド：
  - React
  - TypeScript
  - 状態管理：Zustand
  - DnDライブラリ：dnd-kit
- バックエンド：
  - Node.js
  - TypeScript

## 8. 非機能要件

- パフォーマンス：
  - カードが 10 枚程度までは快適に動作すること
- レイアウト：
  - スマホ・タブレット・PC でのレスポンシブ対応
- テスト対象：
  - Undo/Redo ロジック（履歴の整合性）
  - 永続化（リロード後も位置・内容が保持されること）
  - 基本操作（追加・編集・削除が期待通り動作すること）
